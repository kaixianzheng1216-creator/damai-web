# 大麦网 Web 端项目开发规范 (Version 2.0)

本规范旨在整合 Vue 3, Tailwind CSS v4, TDesign 等最新技术选型，为团队提供统一、高效、高质量的开发指南。

---

## **1. 技术栈概览 (Modern Stack)**

- **框架核心**: [Vue 3.5+](https://vuejs.org/) (Composition API + `<script setup>`)
- **构建工具**: [Vite 6+](https://vitejs.dev/)
- **语言标准**: [TypeScript 5+](https://www.typescriptlang.org/) (Strict Mode)
- **UI 体系**: [TDesign Vue Next](https://tdesign.tencent.com/vue-next/overview) (组件库) + [Tailwind CSS v4](https://tailwindcss.com/) (原子化样式引擎)
- **状态管理**: [Pinia 3+](https://pinia.vuejs.org/) (Store 模式)
- **路由管理**: [Vue Router 4](https://router.vuejs.org/)
- **网络通信**: [Axios](https://axios-http.com/) (统一拦截与类型安全)
- **工程化工具**: [ESLint 10+](https://eslint.org/), [Prettier 3+](https://prettier.io/), [Oxlint](https://oxlint.dev/) (极致 lint 速度)

## **2. 项目架构与命名 (Architecture & Naming)**

### **2.1 目录结构**

```text
src/
├── api/            # 接口定义 (按业务模块划分)
├── assets/         # 静态资源 (images, fonts)
│   ├── css/        # 全局样式 (main.css 定义 @theme)
├── components/     # 公共组件
│   ├── ui/         # 基于 TDesign/Tailwind 封装的基础 UI
│   └── business/   # 业务逻辑组件
├── composables/    # 逻辑复用 (Hooks)
├── router/         # 路由树与守卫
├── stores/         # 状态管理 (Options/Setup Store)
├── types/          # 全局 TS 类型声明 (.d.ts)
├── views/          # 视图/页面 (与路由对应)
├── utils/          # 纯函数工具
├── App.vue         # 根组件
└── main.ts         # 入口
```

### **2.2 命名金律**

- **组件命名**: `PascalCase` (如 `UserCard.vue`)。必须包含两个及以上单词。
- **目录命名**: `kebab-case` (如 `user-profile/`)。
- **变量/函数**: `camelCase` (如 `getUserInfo`)。
- **常量**: `UPPER_SNAKE_CASE` (如 `API_BASE_URL`)。
- **Prop 定义**: 模板中使用 `kebab-case` (如 `:user-id`)，TS 中定义使用 `camelCase` (如 `userId`)。

## **3. 样式体系规范 (Tailwind 4.0 + TDesign)**

### **3.1 Tailwind 4.0 CSS-First 模式**

- **配置位置**: 废弃 `tailwind.config.js`。所有配置均在 `src/assets/main.css` 中通过 `@theme` 块完成。
- **主题令牌**:
  ```css
  /* src/assets/main.css */
  @import 'tailwindcss';
  @theme {
    /* 引用 TDesign 变量实现主题同步 */
    --color-brand: var(--td-brand-color);
    --color-brand-hover: var(--td-brand-color-hover);
    --radius-primary: 8px;
  }
  ```
- **自定义工具类**: 使用 `@utility` 定义，而非传统 CSS。
  ```css
  @utility flex-center {
    display: flex;
    align-items: center;
    justify-content: center;
  }
  ```

### **3.2 TDesign 协同指南**

- **原子类优先**: 间距 (`p-4`, `m-2`)、布局 (`flex`, `grid`)、颜色 (`text-brand`) 优先使用 Tailwind。
- **组件微调**: 使用 Tailwind 4 的 `!` 前缀或行内样式覆盖 TDesign 默认样式。
- **响应式**:
  - 容器级布局：使用 TDesign `t-col` 的响应式属性。
  - 组件内局部：使用 Tailwind 的 `sm:`, `md:`, `lg:` 前缀及 v4 的容器查询 `@md:`。

## **4. Vue 开发最佳实践 (Performance & DX)**

### **4.1 脚本编写规范**

- **必须使用 `<script setup lang="ts">`**。
- **Props 定义**: 推荐使用接口定义 + 编译器宏。
  ```typescript
  const props = defineProps<{
    userId: string
    status?: 'active' | 'inactive'
  }>()
  ```
- **解构响应式**: 必须使用 `toRefs` 或 `storeToRefs` (Pinia)。

### **4.2 性能与优化**

- **计算属性**: 严禁在模板中写复杂的逻辑表达式，必须使用 `computed`。
- **副作用管理**: 所有的定时器、全局事件监听、WebSocket 必须在 `onUnmounted` 中手动清理。
- **懒加载**: 页面级组件必须通过 `defineAsyncComponent` 或 Vite 的动态 import 进行懒加载。

## **5. API 与数据流 (Data Management)**

- **类型安全**: 每个 API 函数必须定义 `Request` 和 `Response` 类型。
- **Mock 驱动**:
  - 开发阶段在 `mock/` 编写数据。
  - 接口命名必须遵循 RESTful 风格。
- **状态同步**: 跨页面数据流转必须使用 Pinia，单页面状态建议使用 `ref/reactive` 配合 `provide/inject`。

## **6. 代码质量与 Git 规范 (Quality Control)**

### **6.1 代码自查 (Checklist)**

- [ ] 是否存在未使用的变量或组件引用？
- [ ] 所有的 API 错误是否都有用户友好的反馈 (`Message.error`)？
- [ ] 是否在 CSS 中定义了不必要的硬编码数值 (Magic Numbers)？
- [ ] 是否为关键逻辑编写了必要的注释？
- [ ] 样式是否优先使用 Tailwind 4 的原子类？

### **6.2 Git 提交**

- 格式：`<type>(<scope>): <subject>`
- 类型：`feat` (新功能), `fix` (修补), `docs` (文档), `refactor` (重构), `chore` (辅助任务)。

---

## **7. 快速工作流 (Quick Start)**

1. **配置主题**: 修改 `main.css` 中的 `@theme`。
2. **定义接口**: 在 `api/` 编写 TS 函数。
3. **构建 UI**: TDesign 组件打骨架 -> Tailwind 原子类填细节。
4. **注入逻辑**: 编写 `composable` 或 `store`。
5. **Lint 检查**: 运行 `npm run lint`。
