# 开发规范

## 核心技术指南

### 1. Vue 3 规范

- **组合式 API**: 强制使用 `<script setup>` 语法，统一使用 `lang="ts"`。
- **Props/Emits**: 使用 `defineProps` 和 `defineEmits` 并声明详细的类型。
- **生命周期**: 优先使用 `onMounted` 等生命周期钩子处理副作用。
- **Ref vs Reactive**: 基础类型使用 `ref`，复杂的对象状态建议也使用 `ref`（更一致），若确实需要响应式对象则使用 `reactive`。

### 2. TypeScript 规范

- **严格模式**: 开启严格模式，避免使用 `any`。
- **接口定义**: 优先使用 `interface` 定义 API 响应和组件 Props。
- **类型文件**: 全局类型放在 `src/types/`，局部类型直接在组件内定义或放在同级 `types.ts`。

### 3. UI 与样式规范

- **设计风格**:
  - **白底黑橙**: 坚持简约风格，背景统一白色，文字黑色，使用品牌橙色 (`primary`) 作为点缀和交互高亮。
- **shadcn-vue**: 优先使用 `src/components/ui` 下的基础组件。如果需要新组件，请通过 `npx shadcn-vue@latest add [component-name]` 添加。
- **Tailwind CSS**:
  - 尽量避免编写原生的 CSS，优先使用 Tailwind utility classes。
  - 使用 `cn` (clsx + tailwind-merge) 处理动态类名。
  - 响应式设计：遵循移动优先原则，使用 `sm:`, `md:`, `lg:` 等前缀。

### 4. API 请求规范

- **Axios 封装**: 统一使用 `src/api/request.ts` 进行请求。
- **TanStack Vue Query**: 所有的异步数据获取（Query）和数据提交（Mutation）必须通过 `useQuery` 或 `useMutation`。
- **拦截器**: 在 `request.ts` 中处理 Token 注入和统一错误响应。

### 5. 目录与命名规范

- **组件**: 帕斯卡命名法（PascalCase），例如 `LoginForm.vue`。
- **文件/目录**: 小写横杠命名法（kebab-case），例如 `user-profile/index.vue`。
- **变量**: 小驼峰命名法（camelCase）。
- **常量**: 全大写下划线（SNAKE_CASE），如 `API_BASE_URL`。

### 6. Git 提交规范

- 提交前会自动运行 `lint-staged` 进行代码检查和格式化。
- 提交信息建议遵循 [Conventional Commits](https://www.conventionalcommits.org/):
  - `feat`: 新功能
  - `fix`: 修复
  - `docs`: 文档变更
  - `style`: 代码格式（不影响功能）
  - `refactor`: 重构
  - `perf`: 性能优化
  - `test`: 测试
  - `chore`: 构建过程或辅助工具的变动

### 7. 开发流程

1. `npm install` 安装依赖。
2. 在 `src/api/` 定义 API 接口。
3. 在 `src/components/ui/` 检查并添加所需的 shadcn 组件。
4. 在 `src/views/` 编写页面逻辑。
5. 提交代码前确保 `npm run lint` 通过。
