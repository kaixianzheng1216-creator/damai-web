# 工单聊天系统 — 前端接入文档

## 速查

| 项目                | 值                                    |
| ------------------- | ------------------------------------- |
| 网关地址            | `http://localhost:18080`              |
| WS 地址             | `ws://localhost:18080/ws/order/chat`  |
| 用户 Token Header   | `Authorization-User: Bearer {token}`  |
| 管理员 Token Header | `Authorization-Admin: Bearer {token}` |
| 统一响应            | `code` 为字符串 `"0"` 表示成功        |
| 分页默认            | `page=1` `size=10` `maxSize=100`      |

---

## 一、鉴权

### HTTP

| Header                | 值                                             |
| --------------------- | ---------------------------------------------- |
| `Authorization-User`  | `Bearer {token}` — 用于 `/api/*/front/**` 接口 |
| `Authorization-Admin` | `Bearer {token}` — 用于 `/api/*/admin/**` 接口 |

### WebSocket

STOMP CONNECT 帧需要携带一个原生 header：

| Header  | 值                                           |
| ------- | -------------------------------------------- |
| `token` | 登录接口返回的 token 原值（**不加 Bearer**） |

后端依次以 `StpKit.USER` 和 `StpKit.ADMIN` 验证该 token，任意一种类型通过即可建立连接，连接后身份写入 `StompPrincipal(name=loginId, accountType="user"|"admin")`。

## 二、HTTP 接口

### 2.1 用户端

| 方法   | 路径                                      | 说明                   |
| ------ | ----------------------------------------- | ---------------------- |
| `GET`  | `/api/order/front/work-orders/my`         | 我的工单列表（分页）   |
| `GET`  | `/api/order/front/work-orders/{id}`       | 工单详情（含回复历史） |
| `POST` | `/api/order/front/work-orders/{id}/close` | 关闭工单               |

### 2.2 管理端

| 方法   | 路径                                      | 说明                   |
| ------ | ----------------------------------------- | ---------------------- |
| `GET`  | `/api/order/admin/work-orders`            | 全部工单列表（分页）   |
| `GET`  | `/api/order/admin/work-orders/{id}`       | 工单详情（含回复历史） |
| `POST` | `/api/order/admin/work-orders/{id}/close` | 关闭工单               |

### 2.3 已移除的接口

以下接口已被删除，**发送消息只能通过 WebSocket**：

| 方法   | 原路径                                      | 状态      |
| ------ | ------------------------------------------- | --------- |
| `POST` | `/api/order/front/work-orders/{id}/replies` | ❌ 已移除 |
| `POST` | `/api/order/admin/work-orders/{id}/replies` | ❌ 已移除 |

历史消息通过 `GET /{id}` 详情接口的 `replies` 字段获取，新增消息通过 WebSocket 实时收发。

### 2.4 列表请求

| 参数        | 类型   | 默认      | 说明                                 |
| ----------- | ------ | --------- | ------------------------------------ |
| `page`      | long   | 1         | 页码                                 |
| `size`      | long   | 10        | 每页条数（上限 100）                 |
| `status`    | int    | 不传=全部 | 筛选：0 待处理 / 1 处理中 / 3 已关闭 |
| `sortField` | string | —         | 排序字段                             |
| `sortOrder` | string | —         | asc / desc                           |

## 三、WebSocket（STOMP）

### 3.1 连接信息

| 项       | 值                                   |
| -------- | ------------------------------------ |
| 端点     | `ws://localhost:18080/ws/order/chat` |
| 应用前缀 | `/app`                               |
| 广播前缀 | `/topic`                             |

### 3.2 STOMP 地址

| 操作     | 目标                                        |
| -------- | ------------------------------------------- |
| 发送消息 | `SEND /app/chat.send`                       |
| 接收消息 | `SUBSCRIBE /topic/work-order/{workOrderId}` |

### 3.3 发送格式（客户端 → 服务端）

```json
{
  "workOrderId": 456,
  "content": "你好"
}
```

仅此两个字段，其余由后端从连接身份中自动填充。

### 3.4 接收格式（服务端 → 客户端）

正常消息：

```json
{
  "type": "CHAT",
  "workOrderId": 456,
  "content": "你好",
  "senderType": "USER",
  "senderId": 1001,
  "timestamp": 1714650123456
}
```

错误消息（工单不存在 / 参数缺失 / 持久化失败）：

```json
{
  "type": "ERROR",
  "workOrderId": 456,
  "content": "工单不存在",
  "senderId": 1001,
  "timestamp": 1714650123456
}
```

字段：

| 字段          | 类型   | 说明                             |
| ------------- | ------ | -------------------------------- |
| `type`        | string | `"CHAT"` 聊天 / `"ERROR"` 错误   |
| `workOrderId` | long   | 工单 ID                          |
| `content`     | string | 消息内容                         |
| `senderType`  | string | `"USER"` 用户 / `"ADMIN"` 管理员 |
| `senderId`    | long   | 发送者 ID                        |
| `timestamp`   | long   | Unix 毫秒时间戳                  |

## 四、枚举值对照

| 域            | 值        | 标签     | 出现位置               |
| ------------- | --------- | -------- | ---------------------- |
| 状态          | `0`       | 待处理   | HTTP `status`          |
|               | `1`       | 处理中   |                        |
|               | `3`       | 已关闭   |                        |
| 类型          | `1`       | 交易异常 | HTTP `type`            |
|               | `2`       | 用户投诉 |                        |
|               | `3`       | 其他     |                        |
| 发送者 (HTTP) | `0`       | 用户     | `replies[].senderType` |
|               | `1`       | 管理员   |                        |
| 发送者 (WS)   | `"USER"`  | 用户     | WS `senderType`        |
|               | `"ADMIN"` | 管理员   |                        |
| 消息类型 (WS) | `"CHAT"`  | 聊天     | WS `type`              |
|               | `"ERROR"` | 错误     |                        |
| 关闭者        | `0`       | 未关闭   | `closedByType`         |
|               | `1`       | 用户     |                        |
|               | `2`       | 管理员   |                        |

## 五、行为说明

- **发送消息唯一入口是 WebSocket**：`POST /front/work-orders/{id}/replies` 和 `POST /admin/work-orders/{id}/replies` 已删除，不可用
- **工单初始内容 `content`** 是用户创建工单时填写的第一条消息，与 `replies` 共同构成完整对话
- **纯广播模型**：订阅了 `/topic/work-order/{id}` 的所有客户端都会收到该工单的每条消息（用户端 + 所有在线管理端），没有「指定管理员」的概念
- **关闭后不可再回复**：已关闭的工单发送消息会收到 `type: "ERROR"` 的广播
- **重连不同步**：WS 断连期间的聊天消息不会补推，需重新调用详情接口获取历史
